# Eduker RFCs

A Request for Comments (RFC) is a short proposal for a non-trivial change. RFCs live in this folder as standalone notes; each one has its own status, owner, and discussion thread.

## How to write an RFC

1. Copy `90-Templates/RFC.md` to `10-Projects/Eduker/RFCs/<short-slug>.md`.
2. Fill it in. Keep it short — 1 page is plenty for most RFCs.
3. Add it to the index below with status `proposed`.
4. Notify the relevant reviewers. Discussion happens in the Paperclip issue or PR linked from the RFC.
5. Move it to `accepted` or `rejected` and update the index.

> [!tip] When to write an RFC
> - Choosing a stack, library, or service.
> - Schema changes that touch more than one table.
> - Cross-cutting changes (auth, logging, deploy).
> - Anything where "we should have asked first" is a likely postmortem.
