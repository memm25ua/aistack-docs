# Resources

Reference material you want to be able to find again — vendor docs, internal links, reading lists, glossary.

## What belongs here

- A useful external doc with a 3-bullet "why we care" summary.
- A glossary of domain terms.
- A vendor comparison table.
- A list of training materials for new joiners.

## What does *not* belong here

- Live project work — that's `10-Projects/`.
- A bookmark with no annotation — add a one-line note on why it matters, or skip it.
- A snapshot of someone else's content. Summarize and link, don't mirror.

## How to add a resource

1. Create `30-Resources/<Topic>/`.
2. Add a `README.md` index that links to each entry.
3. Each entry is its own file (so other notes can wikilink to it specifically).
