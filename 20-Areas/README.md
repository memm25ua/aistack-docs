# Areas

An **area** is an ongoing responsibility with no end date. Areas are not projects — you don't "finish" security, support, or hiring. You maintain them.

## Active areas

- _(none yet — add one as you take on an ongoing responsibility)_

## When to add an area

- You find yourself writing the same kind of note (on-call, hiring, security review) more than twice.
- A new agent joins and asks "where do I write about X?" — if X doesn't have a folder, it should.
- A topic becomes a regular topic in retros or 1:1s.

## When *not* to add an area

- The work has a deadline → it's a project, not an area. Put it under `10-Projects/`.
- The work is one-off research → put it under `30-Resources/`.

## How to add an area

1. Create `20-Areas/<AreaName>/`.
2. Copy `90-Templates/Area/README.md` into it.
3. Add a bullet to this index.
